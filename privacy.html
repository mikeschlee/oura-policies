Privacy Policy

Personal Oura Data Export · Effective July 21, 2026

Purpose and scope

Personal Oura Data Export is a private, single-user tool operated by Mike Schlee. It retrieves the operator's own Oura data to create a personal data archive and offline report. It is not offered to the public and does not accept data from other users.

Data collected

With the operator's explicit OAuth consent, the tool may retrieve Oura profile information and health and wellness data covered by the scopes selected on Oura's consent screen, including daily summaries, sleep, activity, readiness, heart rate, workouts, tags, sessions, and blood oxygen information.

How data is used

Data is used only to build the operator's personal archive, run personal queries, and generate a private offline report. It is not used for advertising, marketing, profiling other people, sale, or training machine-learning models.

Storage and disclosure

OAuth credentials and tokens are stored in access-controlled local secret storage. Retrieved data is stored locally in a private database. A generated report may be copied to the operator's private Google Drive account. Data is not sold or disclosed to other users or unrelated third parties. Infrastructure providers may process data only as needed to provide private storage and transfer services.

Retention and deletion

Data is retained until the operator deletes the local database, generated reports, backups, and any private Google Drive copies. OAuth access can be revoked through Oura, after which the tool will no longer retrieve new data. Because the operator is the only user and data subject, deletion requests are carried out directly by the operator.

Security

Reasonable safeguards are used, including separate secret storage, restricted file permissions, controlled network egress, and private storage destinations. No method of storage or transmission can be guaranteed completely secure.

Changes

This policy may be updated if the tool's data practices change. The effective date above will be revised when material changes are made.

Contact

Questions may be submitted through the policy repository issue tracker. Do not include health data, credentials, or tokens in an issue.
